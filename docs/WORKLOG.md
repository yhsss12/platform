Q：
asyncpg.exceptions.TooManyConnectionsError: sorry, too many clients already
说明 PostgreSQL 连接数已达到上限，新的连接（包括你这次访问）被拒绝。

在日志里你看到的只是 agents/heartbeat 正常返回 200，但在此之前某个请求访问数据库时已经被数据库端拒绝了。

可能原因（结合你这个项目）
后端的多套数据库连接（主库 / jobs 库 / data-assets 库）都走 asyncpg，每个都有自己的连接池；
长时间运行后，如果：
有一些协程异常退出但连接没正确释放，或者
外部其它服务也连着同一个 PostgreSQL
就可能把数据库的 max_connections 打满。
建议的处理步骤（短期）
先重启后端服务（包括你当前的 ./restart.sh），通常会把连接池全部释放，错误会立即消失。
如果你有其他进程也在连同一个 PostgreSQL（比如本地调试脚本、另一个后端实例），可以先关掉那些。
长期需要考虑的
如果这个错误未来频繁出现，建议后续做两件事（需要我们再一起改代码/配置）：

在后端的数据库配置里 显式限制每个连接池的最大连接数，例如 5–10，而不是默认值；
在 PostgreSQL 侧合理调大 max_connections，确保综合所有服务的连接池之和不会轻易打满。
